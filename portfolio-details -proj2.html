Project Details

Deploy a WordPress Website using various AWS services

Project information

  • Category: AWS project
  • AWS Used services: EC2 instances ,NAT gateways ,RDS ,Application load balancer ,Route 53,Auto scaling group ,Certificate manager ,EFS and ec2 bastion host
  • Project date: 15 October,2024

Objective

deploy dynamic website on AWS. Also Architect the way to use reference architecture to host this website using various AWS services and illustrate how they work together.

Amazon web services that used :

  • VPC with public and private subnets
  • Security groups
  • EC2 instances
  • NAT gateways
  • RDS
  • EFS
  • Application load balancer
  • Auto scaling group
  • bastion host

    • VPC Reference Architecture

    We will start by creating a custom 3-tier VPC using VPC reference architecture above. In a 3-tier reference architecture, our infrastructure is divided into 3 tiers.
    Tier 1 — Two public subnet which will hold the resources such as NAT gateway, load balancer, and bastion host
    Tier 2 — Two private subnet contains ec2 with web servers
    Tier 3 — another private subnet which will hold our database (RDS)
    We will duplicate these subnets across Two availability zones for high availability and fault tolerance,If an AZ does down for some reason, the application has the ability to automatically scale resources to another AZ, without affecting the rest of the application tiers

    Part 10: Create an Auto Scaling Group

    We are going to set up an auto-scaling group to automatically create and manage web servers in the private
    application subnet.

    First, we need to terminate the EC2 instances that were created manually. Navigate to the "Instances" section
    within the EC2 Dashboard. Select both instances named Webserver AZ1 and Webserver AZ2, click "Instance State,"
    and choose "Terminate." These instances will start terminating.

    Next, we need to create our Launch Templates. These templates will include the configuration details for the EC2
    instances that the auto-scaling group will use to launch new instances,I have created one before so i gust choose it

    choose our vpc and private subnets .

    For the Load Balancing section, we'll link our auto scaling group to the Application Load Balancer (ALB)
    that we set up earlier. Under Load Balancing, make the following selections:

  • Attach to an Existing Load Balancer: Select this option to connect your auto scaling group to
    a pre-existing load balancer.
  • Choose from Your Load Balancer Target Groups: From the available list, pick your existing load
    balancer target group.


    • We want our auto scaling group to maintain two EC2 instances and the ASG can scale down to a minimum 1
    instance or a maximum of 4 instances, click Next ,use another words in previous sentences Let's configure our
    auto scaling group to maintain a steady state of two EC2 instances. The auto scaling group will have the flexibility
    to scale down to as few as one instance or up to a maximum of four instances. Once these settings are in place,
    proceed by clicking "Next."

    You should see two instances running, which were launched by the Auto Scaling Group (ASG). These instances
    can be identified by the tag names we assigned to them (ASG-Webserver). Let's wait for them to pass the Status check.

    Next, let's verify that the Health status of the Registered targets is healthy. You can do this by navigating
    to Target Groups under the Load Balancing section
